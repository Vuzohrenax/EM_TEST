# Тестовый веб-проект с проксированием через Nginx

Этот проект использует Docker Compose для запуска бэкенда (`web-app`), скрытого внутри изолированной сети, и прокси-сервера `Nginx`, который выступает как единая точка входа.

---

## 🚀 Как запустить проект

Для запуска вам понадобятся установленные **Docker** и **Docker Compose**.

1. **Клонируйте репозиторий** и перейдите в папку проекта.
2. **Соберите и запустите** контейнеры одной командой:
   ```bash
   docker compose up -d --build
   ```
   *(Флаг `-d` запустит проект в фоновом режиме, а `--build` соберет свежий образ бэкенда).*

---

## 🔍 Как проверить результат

Контейнер бэкенда не публикует порты наружу напрямую, но доступен через Nginx.

1. **Откройте браузер** и перейдите по адресу: [http://localhost](http://localhost) (или `http://127.0.0.1`).
2. **Альтернативный вариант** — проверка через терминал с помощью утилиты `curl`:
   ```bash
   curl http://localhost
   ```
Если всё настроено правильно, вы увидите ответ от вашего веб-приложения.

## 🛠️ Как работает эта схема

Проект использует концепцию обратного проксирования (Reverse Proxy) и безопасной изолированной сети:

[ Пользователь / Браузер ] > localhost:80 > nginx_proxy_container(Доступен из внешнего мира) > (Сеть test_network) > test_container(Скрыт от внешнего мира)


* **Безопасность:** Бэкенд (`web-app`) не имеет опубликованных наружу портов (`ports:` отсутствует). Злоумышленник не может подключиться к нему напрямую из интернета.
* **Изоляция:** Оба контейнера подключены к одной виртуальной сети `test_network`. Они могут общаться друг с другом по внутренним именам сервисов.
* **Маршрутизация:** Nginx принимает все внешние запросы на порту `80`, сверяет их со своими правилами в `nginx.conf` и перенаправляет внутрь сети на адрес `http://web-app` (или конкретный порт бэкенда).
* **Поддержка SELinux:** В конфигурации Compose используется флаг `:z` для монтирования файлов Nginx. Это предотвращает ошибки доступа на таких системах, как Fedora, RedHat или CentOS.
